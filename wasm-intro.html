WebAssembly (Wasm) represents a significant step forward in web development, allowing us to run high-performance code written in languages like Rust directly in the browser. This post will guide you through the basics of setting up a Rust + WebAssembly project.

Why Rust + WebAssembly?

Rust's memory safety guarantees, zero-cost abstractions, and excellent WebAssembly tooling make it an ideal choice for Wasm development. You get near-native performance while maintaining the safety that Rust provides.

Setting Up Your First Project

We'll start with a simple example that adds two numbers - the "Hello World" of WebAssembly:

use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

This simple function demonstrates the basic pattern: annotate your Rust functions with #[wasm_bindgen] to make them callable from JavaScript.
